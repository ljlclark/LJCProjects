Code Backup Profiles

A code backup action is defined by a backup profile. These profiles are defined in the file "BackupProfiles.txt". This file must be in the same folder as the backup program.

Backup profiles are defined by Name and Value pairs separated with a comma.

The location from which the code will be retrieved is called the "Source" and the location to which the code will be copied is called the "Target".

A backup profile starts with the name "Backup" followed by a comma and the profile name.

The "SourceRoot" name is set to the "Source" root path. For example: "C:\Users\User\Documents\Visual Studio 2022".

The "SourceCodeline" is set to the "Source" codeline folder name. For example: "LJCProjectsDev".

The "TargetRoot" name is set to the "Target" root path. For example: "C:\Users\User\Documents\Visual Studio 2022".

The "TargetCodeline" is set to the "Target" codeline folder name. For example: "LJCProjects".

ChangesFileSpec is set to the name of the "Changes" file.

IncludeFilters is set to a list of file patterns that identify the files that will be backed up. The file patterns are separated with a vertical bar (pipe) character. For example: *.cs|*.sln
The set of files that match each file pattern will be retrieved and processed in the order that they are defined in the IncludeFilters value. This will be reflected in the order of commands in the "Changes" file.

SkipFiles is set to a list of specific files that will be excluded from the backup. For example: Changes.txt|BackupLog.txt|MissingFolders.txt

SkipSubFolders is set to common folders that are to be excluded from the backup. For example: .vs|obj

SkipCodePathspec is set to the filespec of the file containing code paths that are to be skipped in the backup.
This is useful where not all folders from a codeline are to be promoted.
A code path is the file path that appears after the codeline folder name. For example: in C:\Users\User\Codelines\Codeline\SolutionName\ProjectName
Root = C:\Users\User\Codelines
Codeline = Codeline
Codepath = SolutionName\ProjectName
In this case if the file defined in SkipCodePathspec contains the code path "SolutionName\ProjectName", it will not be backed up.

IncludeMissingTargetFolders is set to "true" or if missing, is assumed "false". If "true" then any missing target folders are created and backed up even if they are in the SkipCodePathspec file.
