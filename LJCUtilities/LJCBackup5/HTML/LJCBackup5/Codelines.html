Code Lines

Related source code is usually kept under a parent folder. This documentation refers to this folder as the "Codeline". A codeline may contain code related to an application, framework, programming language or other classification.

Code lines are organized under a common or "root" folder. For example: C:\Users\User\Codelines. If one is using Microsoft Visual Studio, the standard location for source code is something like: C:\Users\User\Documents\Visual Studio 2022\Projects. Where "Projects" is the "Codeline" and C:\Users\User\Documents\Visual Studio 2022 is the codeline "root".

On a developers machine, it is common for there to be two codelines for each code base; one that is under version control and one for current development. Changes are made in the development codeline and, when complete, are promoted to the version control codeline. The version control codeline is then promoted to the version control system (VCS) shared codeline using the VCS software.

The LJC C# codelines are: version control codeline, "LJCProjects" and development codeline, "LJCProjectsDev". Both of these codelines are in the root C:\Users\User\Documents\Visual Studio 2022.
