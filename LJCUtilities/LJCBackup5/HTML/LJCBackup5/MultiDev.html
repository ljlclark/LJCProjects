Multi-Developer Environments

A multi-developer environment requires a version of the following steps when making a code change. These steps require a shared version control code line which is managed by a Version Control System (VCS).

1. Verify all previous developer changes have been tested, promoted and checked in correctly.
2. Refresh the developers version control codeline to get the current code.
3. Use the VCS to "Checkout" the files to be modified.
4. Refresh the developers development codeline to set the starting point.
5. The developer may want to verify the behavior to be fixed at this point.
6. Code and "Dev Test" the development changes.
7. Refresh the developers version control codeline to get any changes that happened while the developer was coding.
8. Copy the development code changes to the local version control codeline. The local version control line should only contain source code. It should not contain executable files or other files that are generated from the source code.
9. Refresh the developers development codeline and perform an integration test.
10. Use the VCS to update the shared codeline.
