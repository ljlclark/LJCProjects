Multi-Developer Environments

A multi-developer environment requires a version of the following steps when making a code change. These steps require a shared version control code line which is managed by a Version Control System (VCS).

1. Verify all previous developer changes have been tested and promoted correctly.
2. Refresh the developers version control codeline to get the current code.
3. Use the VCS to "Checkout" the files to be modified.
3. Refresh the developers development codeline to set the starting point.
4. Code and "Dev Test" the development changes.
5. Refresh the developers version control codeline to get any changes that happened while the developer was coding.
6. Copy the development code changes to the local version control codeline.
7. Integration test the local version control line.
8. Use the VCS to update the shared codeline.
