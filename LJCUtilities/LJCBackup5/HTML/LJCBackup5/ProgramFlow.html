Code Backup Program Steps

The backup program contains several steps. These are described here at a high level.

First the program creates a "Changes" file. This contains "Copy" commands with the names of the files that will be copied to the backup target. These are files that were added or changed in the backup source by the developer.

It also contains "Delete" commands with the names of the files that will be deleted from the backup target. These are files that are still in the backup target but have been delete in the backup source by the developer.

The developer can modify the "Changes" file before the next step. This may include actions such as removing deletes that the developer wants to verify before the actual backup is performed.

The backup program then reads and applies the change commands contained in the "Changes" file.

Note:
The LJC Code Backup program does not contain code "Merge" functionality. A code "Merge" may be performed manually by the developer before promoting code to the version control codeline.
